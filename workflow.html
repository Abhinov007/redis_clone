sequenceDiagram participant RP as replica.js participant M as master.js (server.js) Note over RP: State: WAIT_WELCOME RP ->> M : PING M -->> RP : +PONG Note over RP: State: REPLCONF_PORT RP ->> M : REPLCONF listening-port <port> M -->> RP : +OK Note over RP: State: REPLCONF_CAPA RP ->> M : REPLCONF capa psync2 M -->> RP : +OK Note over RP: State: WAIT_SYNC_RESPONSE alt First sync (no saved replid) RP ->> M : PSYNC ? -1 M -->> RP : +FULLRESYNC <replid> <offset> M -->> RP : $<size>\r\n<RDB bytes> RP ->> RP : loadRDBFromBuffer(data) Note over RP: State: STREAMING else Reconnect (saved replid + offset) RP ->> M : PSYNC <saved_replid> <saved_offset> alt offset in backlog M -->> RP : +CONTINUE <replid> M -->> RP : <backlog bytes from offset> Note over RP: Partial sync — no RDB transfer! else offset too old M -->> RP : +FULLRESYNC <replid> 0 M -->> RP : $<size>\r\n<RDB bytes> end Note over RP: State: STREAMING end loop Every 1 second (heartbeat) RP ->> M : REPLCONF ACK <replicaOffset> M ->> M : update replica.offset end loop Write commands from clients M -->> RP : <RESP-encoded command> RP ->> RP : handleCommand(args) — apply to replica DB end

🔒 MULTI / EXEC Transaction Flow

📦 Module Responsibilities

server.js

redis_clone/server.js

TCP server entry point. Accepts connections, detects protocol, dispatches commands, manages transaction state, orchestrates startup.

Core

RESPParser.js

redis_clone/RESPParser.js

Stateful RESP protocol parser. Buffers partial TCP chunks and emits complete command arrays.

Protocol

protocol/resp.js

redis_clone/protocol/resp.js

RESP encoding helpers: respSimpleString, respError, respBulk, respArray, respInteger, toRESP.

Protocol

commands/command.js

redis_clone/commands/command.js

Implements all Redis commands: String, List, Hash, Pub/Sub, System. Routes to storage and messaging layers.

Core

storage/database.js

redis_clone/storage/database.js

In-memory Map with typed entries (string / list / hash). Provides get/set/delete/getAll APIs.

Storage

storage/expiry.js

redis_clone/storage/expiry.js

TTL management. Stores expiry timestamps; lazy deletion triggered on key access.

Storage

persistence/aof.js

redis_clone/persistence/aof.js

Append-Only File logging. Every write command is appended; replayed on startup to restore incremental state.

Persistence

persistence/rdb.js

redis_clone/persistence/rdb.js

JSON-based RDB snapshots of the full database. Saved after every write; loaded on startup and during replica full sync.

Persistence

replication/master.js

redis_clone/replication/master.js

Master replication: tracks replica sockets, manages 1 MB replication backlog, handles REPLCONF/PSYNC, propagates write commands.

Replication

replication/replica.js

redis_clone/replication/replica.js

Replica state machine: PING → REPLCONF → PSYNC → stream. Auto-reconnects after 3 s. Sends ACK heartbeat every 1 s.

Replication

messaging/pubsub.js

redis_clone/messaging/pubsub.js

Pub/Sub engine. channel → Set<socket> map. Handles SUBSCRIBE, UNSUBSCRIBE, PUBLISH, and cleanup on socket close.

Messaging

client.js

redis_clone/client.js

Simple CLI client. Connects to server, sends commands from stdin, receives and prints responses.
